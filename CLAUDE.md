# Before implementing a feature

- Scan the code base.
- Take a look at what needs to change for this feature to be implemented.
- Create a new file in `documents` (prefixed with index, counting up)
  - In it, make a list of all TODOs relevant for the feature.
  - Group these TODOs into milestones.

# After implementing a feature

- Commit your changes to Git.
- Update the markdown files.
  - Check all TODOs that have been done.

# Regularily

- Take a look of the codebase:
  - Are all features marked as done actually implemented?
  - Or is there something missing?
  - Are there bugs?
  - Things we can improve?

# Always

- Make use of best practices.
- Write code of very high quality.
- Include tests of everything that is testable.
- Make no mistakes.
